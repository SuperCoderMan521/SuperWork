“在编辑器中打开”菜单放在哪里？

点击任意方案选择。三种方案都会实时检测本机已安装编辑器，并打开整个工作区。

打开工作区
🔷 Visual Studio Code
◼ Cursor
◆ JetBrains WebStorm
↻ 重新检测编辑器

A · 按钮下拉菜单（推荐）

位置与系统资源管理器一致，路径短，不遮挡文件内容。

选择编辑器
🔷 Visual Studio Code
◼ Cursor
◆ JetBrains WebStorm

B · 居中弹窗

选择更醒目，但会遮挡当前文件预览，交互稍重。

搜索编辑器…
🔷 Visual Studio Code
◼ Cursor
◆ JetBrains WebStorm

C · 命令面板

适合安装很多 IDE 的用户，但当前规模下搜索框显得多余。

共同状态设计

首次展开显示“正在检测编辑器…”；检测成功只列出可启动项；未发现时显示“未检测到支持的编辑器”与“重新检测”；启动失败时保留菜单并在对应项下显示错误。
